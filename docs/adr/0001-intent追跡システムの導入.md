# ADR-0001: Intent追跡システムの導入

## ステータス
Accepted

## コンテキスト
エージェントシステムの開発中、各変更の意図を追跡することが困難になっていた。コミットメッセージだけでは「なぜ」その変更を行ったかが分からず、後から理解するのに時間がかかっていた。

## 決定
Worker出力に変更意図（Intent）セクションを追加し、YAMLファイルとして保存する。TUIダッシュボードでIntent・Diff・ADRを統合表示する。

## 理由
1. 変更の「why」を記録することで、将来の理解が容易になる
2. ADRとの連携により、アーキテクチャ決定との関連が明確になる
3. TUI統合により、既存のワークフローを変更せずに確認できる

## 結果
1. Worker出力形式が変更される（後方互換性あり）
2. state/intents/ ディレクトリが追加される
3. PyYAML依存が追加される
